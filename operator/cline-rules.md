# CLINE EXECUTION RULES (GAHWA SYSTEM)

These rules MUST always be followed for every task.

## 1. EXECUTION DISCIPLINE
- Execute ONE task at a time
- Never combine multiple fixes in one step
- Stop immediately after completing the task

## 2. NO HIDDEN WORK
- Do not continue after finishing a step
- Do not infer missing requirements
- Do not expand scope

## 3. FILE CHANGE RULE
- Modify only files directly related to the issue
- Maximum 1–3 files per task

## 4. OUTPUT RULE
After each change, print:
- file changed
- exact change made
- confirmation of completion
Then STOP.

## 5. FAILURE HANDLING
If unclear, choose smallest safe fix only.

## 6. CORE PRINCIPLE
Cline is a deterministic executor, not a planner.
